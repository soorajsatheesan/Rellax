# Employer Signup Onboarding Flow

## Auth Decision

**Recommended Auth:** WorkOS AuthKit + Convex

**Why this is the best fit for the current stack**
* The current app is `Next.js 16 + Convex` and does not yet have an auth provider installed.
* WorkOS AuthKit is B2B-focused and maps well to the product's company-first onboarding model.
* Convex has official integration guidance for WorkOS AuthKit in existing applications.
* WorkOS supports organization-aware authentication and is a better long-term fit for employer-managed employee access.
* Convex Auth is currently documented as beta, so it is not the safest default for this production-facing flow.

**Implementation direction**
* Use WorkOS AuthKit for identity, session management, route protection, and organization-aware authentication.
* Use Convex for app data: companies, memberships, onboarding state, dashboard completion state.
* Treat the first signed-up employer user as the company admin/manager.

## 1. Epic (The "Big Picture")

**ID:** `e1`

**Title:** `[EPIC] - Employer Company Signup and Onboarding`

**1. Executive Summary**
* **Objective:** Enable a company manager to create a company account, complete initial company onboarding, and land in the employer dashboard with a clear prompt to finish the company profile.
* **Business Value:** This creates the first production onboarding path for employer admins, which is the entry point for company-managed employee onboarding in RELLAX.

**2. Scope**
* **In Scope:**
  * Company-first sign-up flow
  * Fields: company name, work email, password, country, terms acceptance
  * Employer account creation and company admin setup
  * Redirect to company onboarding after successful sign-up
  * Company onboarding form with logo/profile image upload
  * Redirect to dashboard after onboarding
  * Dashboard home CTA to complete missing company profile details such as company URL
  * Auth route protection for employer-only onboarding and dashboard routes
* **Out of Scope:**
  * Employee self-sign-up
  * Employee onboarding flow
  * SSO / enterprise identity providers
  * Billing / subscription setup
  * Bulk employee import
  * Advanced company settings beyond minimum onboarding

**3. Success Metrics (KPIs)**
* Employer account creation completion rate
* Onboarding completion rate after signup
* Time from signup submission to first dashboard load
* Percentage of companies completing company URL/profile completion CTA

**4. Key Deliverables**
* Employer signup page
* WorkOS AuthKit + Convex auth foundation
* Company and membership data model
* Company onboarding page
* Employer dashboard home page with profile completion CTA
* Tests for validation, routing, and core happy path

## 2. User Story (The "Vertical Slice")

**ID:** `s1`

**Title:** `[STORY] - Sign up a company account and complete employer onboarding`

**1. The Narrative (Standard Format)**
> **As a** company manager,
> **I want to** sign up my company, complete the initial setup, and enter the dashboard,
> **So that** I can start configuring onboarding for my employees.

**2. Acceptance Criteria (The "Definition of Done")**
* [ ] **Scenario 1:** Company signup happy path
    * **Given** the user is on the signup page
    * **When** they enter company name, work email, password, country, accept terms, and submit valid data
    * **Then** a company-admin account is created and the user is redirected to the company onboarding page
* [ ] **Scenario 2:** Required field validation
    * **Given** one or more required fields are missing
    * **When** the user submits the signup form
    * **Then** inline validation errors are shown and account creation does not proceed
* [ ] **Scenario 3:** Terms and conditions enforcement
    * **Given** the terms checkbox is not selected
    * **When** the user submits the signup form
    * **Then** signup is blocked with a clear validation message
* [ ] **Scenario 4:** Duplicate or invalid email handling
    * **Given** the email is invalid or already associated with an account
    * **When** the user submits the form
    * **Then** the user sees a safe, user-friendly error message
* [ ] **Scenario 5:** Company onboarding happy path
    * **Given** the employer account has been created
    * **When** the user uploads a company logo/profile image and submits onboarding
    * **Then** the company profile is saved and the user is redirected to the dashboard home page
* [ ] **Scenario 6:** Dashboard completion CTA
    * **Given** the company profile is still incomplete
    * **When** the user lands on the dashboard home page
    * **Then** they see a clear prompt to complete missing fields such as company URL
* [ ] **Scenario 7:** Route protection
    * **Given** an unauthenticated visitor tries to access employer onboarding or dashboard routes
    * **When** the route loads
    * **Then** they are redirected to login/signup appropriately
* [ ] **Scenario 8:** UI/Responsiveness
    * Matches the intended design direction on both Mobile and Desktop

**3. Technical Constraints / Notes**
* Frontend: Next.js App Router
* Backend/data: Convex
* Auth recommendation: WorkOS AuthKit with Convex integration
* Keep the product UX company-first, even if parts of auth/session handling use WorkOS hosted flows
* Persist a company record and a membership/admin relationship for the first employer user
* The signup input label/placeholder should say "Work email", but validation should accept any valid email format for now
* Country should be required and stored during signup
* Terms acceptance must be stored as a timestamp / acceptance event or equivalent auditable field
* Company logo upload needs a safe storage strategy and file validation
* Security considerations:
  * validate all form inputs
  * sanitize stored company fields
  * protect employer-only routes
  * avoid leaking auth/provider errors directly to the UI

**4. Estimations**
* Story Points: 13
* T-Shirt Size: L

## Example Workflow

### Epic: Employer Company Signup and Onboarding
* **Goal:** Allow a company manager to sign up the company, complete onboarding, and enter the employer dashboard.

### Story: Sign up a company account and complete employer onboarding
* **As a** company manager, **I want to** create my company account and finish setup, **So that** I can start onboarding employees.
* **AC:**
    * Given valid signup data -> Create company admin account and redirect to onboarding.
    * Given completed onboarding -> Redirect to dashboard.
    * Given incomplete profile -> Show company profile completion CTA.

### Task: Backend provisioning for employer signup
* **Action:** Create company, company-admin membership, onboarding status, and secure route handling.
* **Details:**
    * Accept `companyName`, `workEmail`, `password`, `country`, `termsAccepted`.
    * Create auth identity through WorkOS AuthKit.
    * Persist company and admin membership in Convex.
    * Redirect to onboarding, then dashboard.
